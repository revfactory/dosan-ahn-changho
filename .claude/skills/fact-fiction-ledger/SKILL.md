# 사실-허구 대장 (fact-fiction-ledger)

소설은 유추와 창작을 허용한다 — 단 **무엇이 사실이고 무엇이 지어낸 것인지를 작품 스스로 알고 있어야 한다.** 이 스킬은 그 장부를 규정한다. 장부가 있어야 ① 검증 사실과 모순되는 창작을 기계적으로 잡고, ② 부분 수정 시 어떤 장면이 어떤 사실에 묶여 있는지 추적하며, ③ 작가의 말에서 독자에게 정직하게 공개할 수 있다. 이 프로젝트의 검증 정직성은 소설에서도 형태만 바꿔 유지된다.

## 1. scene_ledger.json 스키마

`novel/ledger/scene_ledger.json` — 장면당 1레코드, scene-writer가 집필과 동시에 작성하고 fiction-fact-keeper가 검증·확정한다.

```json
{
  "scene_id": "ch07-s2",
  "chapter": 7,
  "time": "1919-06 하순",            // 연표 정밀도 이내로만 특정
  "place": "상하이 프랑스조계",
  "characters": ["안창호", "이광수", "주요한"],
  "fact_anchors": [                   // 이 장면이 딛고 선 검증 사실
    {"id": "evt-provgov-003", "use": "배경사건"},   // use: 배경사건|직접재현|대사소재
    {"id": "clm-0142", "use": "대사소재"}
  ],
  "inferences": [                     // 유추·창작 요소 — 전부 기록
    {"type": "추정대화", "desc": "취임 전야 이광수와의 대화 내용", "basis": "두 사람의 당시 직책·이후 행적에서 개연 유추"},
    {"type": "창작인물", "desc": "여관 주인 노파", "basis": "장면 기능상 필요, 실존 인물 아님"}
  ],
  "adopted_lore": [                   // D등급·기각 전승의 채택 (선택)
    {"id": "clm-0285", "desc": "오렌지 일화", "note": "출전 미상 전승 — 소설적 채택"}
  ],
  "disputed_choice": [                // 상충 사건에서 택한 설 (선택)
    {"id": "evt-shin-021", "choice": "1910-04-07 출발설"}
  ],
  "verified": false                   // fiction-fact-keeper 확정 시 true
}
```

inference type 어휘: `추정대화`(기록된 사건 안의 대사 창작 — 가장 흔함) / `추정장면`(기록 사이의 공백을 잇는 장면) / `창작인물`(허구 단역) / `시간압축`(사건 병합·재배열) / `내면추정`(감정·동기).

## 2. 유추 허용 규칙 — 무엇을 지어내도 되는가

| 허용 | 조건 |
|------|------|
| 기록된 사건 속 대사·내면 | 인물의 검증된 입장·관계와 정합할 것 (network.json 엣지 방향과 모순 금지) |
| 기록 공백을 잇는 장면 | 확정 공백 구간(1879–85, 1916, 1934, 1936 등)은 소설의 기회다 — 단 앞뒤 검증 사건과 인과 모순 금지 |
| 허구 단역 | 실존 인물과 혼동될 이름 금지, ledger 등록 |
| 전승·D등급 일화 채택 | 자유 — 오렌지 일화, 이토 면담, 임종 어록 등은 소설적으로 좋은 재료다. adopted_lore에 기록하면 된다 |
| disputed 양설 중 택일 | 자유 — disputed_choice에 기록. 단 한 작품 안에서 일관되게 |
| 시간 압축·사건 병합 | 연 단위 이내, ledger에 기록 |

## 3. 반사실 금지 목록 — 기계 검사 대상

1. **생몰 위반**: 사망 후 등장·출생 전 등장 금지. network.json의 birth/death가 기준 (예: 이토 히로부미 1909-10 사후 등장 금지, 안수산 1915 출생 전 등장 금지).
2. **시공간 위반**: 검증 연표상 그 시기 다른 대륙에 있는 인물의 대면 금지 (timeline.json period·place 대조 — 예: 1924년 도산은 미주, 상하이 장면에 넣지 말 것).
3. **확정 사실 변경**: A·B등급 확정 날짜·결과의 변경 금지 (105인 판결, 순국 일자, 창립일 등 — 압축은 가능하나 결과를 바꾸지 마라).
4. **실명 인물 명예 창작**: 기록 근거 없는 범죄·배신·추문을 실명 인물에게 부여 금지. 갈등은 conflict 엣지·conflicts.md 기록 범위에서 강화하라. (김희선 귀순처럼 기록된 사실은 사용 가능.)
5. **관계 반전**: 검증 관계망의 유형(동지/갈등/사제)을 뒤집는 설정 금지 — 시기별 변화는 엣지 period를 따른다(신채호: 1910 동지 → 1923 대립).

## 4. 검증 절차 (fiction-fact-keeper)

장 단위 incremental — 장 원고+장면 대장이 도착하면 즉시:
1. **대장 존재**: 본문 장면과 ledger 레코드 1:1 (누락 장면 = 차단).
2. **기계 검사**: 등장 인물 전원을 network 생몰·timeline 동선과 대조(§3-1·2), fact_anchors id 실재 확인, disputed_choice 작품 내 일관성.
3. **의미 검사**: 대사가 검증된 입장과 모순되지 않는가 (예: 도산이 무장투쟁 즉발론을 설파하면 모순 — 단 "흔들리는 장면"은 내면추정으로 가능, 기록된 노선과의 최종 정합 필요).
4. 판정: PASS / 수정 회부(구체 위치·모순 근거·수정 방향). verified=true 갱신은 keeper만 한다.

## 5. 작가의 말 생성

탈고 후 `novel/afterword.md` — ledger 집계로 자동 골격 생성 후 문장화:
- 이 소설이 딛고 선 검증 사실의 규모 (사건 N건·검증 주장 N건)
- 시기 압축·재배열 목록 (주요 건)
- 채택한 전승 목록과 그 출전 상태 ("오렌지 일화는 출전 미상의 전승이다 — 소설은 이를 받아들였다")
- 순수 창작 요소 (허구 인물·추정 대화의 범위)
- 안내: 검증된 기록 전체는 동반 웹사이트에서 볼 수 있다 (URL)

이 공개가 소설의 신뢰를 깎지 않는다 — 오히려 "어디까지가 기록인가"를 아는 독자가 더 깊이 빠진다.

## 산출물 검증

1. ledger 스키마 유효·scene_id 유일·장면 1:1 (스크립트)
2. 반사실 검사 5종 위반 0 (스크립트 — 생몰·동선은 network/timeline 자동 대조)
3. 전 장면 verified=true 후에만 탈고 선언
4. afterword가 ledger 집계와 수치 일치
