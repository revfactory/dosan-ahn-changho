# 사실 검증 프로토콜

Phase 1의 조사 산출물은 양은 많지만 신뢰도가 균질하지 않다. 검증 단계의 목적은 틀린 것을 골라내는 것만이 아니라, 모든 주장에 "얼마나 믿을 수 있는가"라는 꼬리표를 다는 것이다. 등급이 붙은 주장만이 Phase 3에서 안전하게 사용될 수 있다 — 작가는 A·B등급만으로 본문을 쓰고, C등급은 한정 표현과 함께, D등급은 배제한다. 등급이 없으면 작가가 신뢰도를 임의 판단하게 되고, 그것이 바로 오류가 최종 사이트에 도달하는 경로다.

## 1. 주장 단위 분해법

검증의 단위는 문장이 아니라 주장(claim)이다. 한 문장에 여러 주장이 섞여 있으면 각각의 신뢰도가 다를 수 있으므로 분해해야 한다.

예: "안창호는 1908년 평양에 대성학교를 설립하고 교장으로 취임했다"는 문장은 최소 3개 주장이다.

1. 대성학교가 1908년에 설립되었다 — 설립 연도 주장
2. 설립 장소는 평양이다 — 장소 주장
3. 안창호가 교장으로 취임했다 — 직책 주장 (실제로는 표면상 교장이 따로 있었다는 이설이 존재하는 영역)

분해 원칙:

- **하나의 주장 = 하나의 검증 가능한 명제.** "누가-언제-어디서-무엇을" 중 두 요소 이상이 결합되어 있고 각각 독립적으로 틀릴 수 있다면 분해하라.
- **수식어도 주장이다.** "최초의", "유일한", "비밀리에" 같은 수식어는 별도 검증이 필요한 강한 주장이다.
- **분해 후에도 원문 추적 가능해야 한다.** 각 claim은 출처 사건 레코드 id(`event_ids`)를 보존하라.

## 2. 신뢰도 등급 기준표

| 등급 | 기준 | 판정 예시 |
|------|------|----------|
| **A** | 1차 사료가 존재하고 학술 연구도 일치 | "안창호는 1938년 3월 10일 경성제국대학 부속병원에서 사망했다" — 당시 신문 보도(1차)와 모든 학술 전기가 일치. A 부여 |
| **B** | 1차 사료는 미확보이나 신뢰할 만한 학술 출처가 뒷받침 (단일 학술 출처 포함) | "안창호는 리버사이드에서 오렌지 농장 한인 노동자들의 작업 품질 향상 운동을 이끌었다" — 학술 연구들이 서술하나 동시대 1차 기록은 직접 확인 못함. B 부여 |
| **C** | 2차 사료·전기·회고류에만 근거 | "어려서 서당에서 필대은을 만나 신학문에 눈떴다" — 후대 전기류의 서술로만 전해지고 동시대 기록·학술 검증 미확인. C 부여 |
| **D** | 출처 미확인 또는 출처 간 치명적 상충으로 판단 불가 | "안창호가 ○○ 연설에서 정확히 이 문장을 말했다" — 연설문 원문이 미확인이고 인용이 글마다 다름. D 부여 |

등급 판정 시 유의:

- 등급은 주장의 중요도가 아니라 **근거의 강도**다. 사소한 주장도 1차 사료가 있으면 A, 핵심 주장도 출처가 없으면 D다.
- 1차 사료가 있어도 사료 비판(S03) 결과 작성자 편향이 핵심을 좌우하면(예: 일제 신문조서의 진술 내용) A가 아니라 B 이하로 — 사유에 비판 결과를 기록하라.
- 소재 미확인 사료의 2차 인용(S03의 `cited_only`)에 근거한 주장은 A가 될 수 없다 — 재인용은 1차 사료 확인이 아니다.
- 판정 사유(`grade_reason`)는 반드시 적어라. 사유 없는 등급은 재검증이 불가능하고, 등급 자체가 검증 불가 주장이 된다.
- 의심스러우면 낮은 등급으로. 과대 등급은 오류를 본문에 올려보내고, 과소 등급은 보완 조사를 부를 뿐이다 — 두 오류의 비용은 비대칭이다.

## 3. 독립 출처 2개 원칙

등급 부여 전 독립된 출처 2개의 확인을 시도하라. 실패하면 삭제하는 것이 아니라, 실패 사실을 `grade_reason`에 기록하고 단일 출처 기준으로 등급을 매긴다.

핵심은 "독립"의 판별이다. **같은 뿌리에서 나온 인용은 아무리 많아도 1개로 계산한다.** 백과사전 → 블로그 → 언론 기사로 복제된 같은 문장 셋은 출처 1개다. 독립성 판별법:

- 두 출처가 서로를 인용하거나 공통의 제3출처를 인용하면 비독립.
- 표현·수치·오탈자가 동일하면 복제를 의심하라 — 특히 특이한 세부(정확한 날짜, 인원수)가 글자까지 같으면 거의 확실한 복제다.
- 독립성이 확인되는 가장 강한 조합: 성격이 다른 1차 사료 2종(예: 동시대 신문 + 일제 기록), 또는 서로 다른 사료군에 기반한 학술 연구 2편.

이 원칙이 필요한 이유: 도산 관련 서술은 소수의 초기 전기(특히 이광수 전기)가 뿌리인 경우가 많아, 인용 횟수가 신뢰도로 착각되기 쉽다.

## 4. 상충 처리

출처 간 진술이 다를 때의 절대 원칙은 **삭제 금지**다. 상충 데이터를 지우면 "상충이 있었다"는 정보 자체가 사라지고, 후속 연구가 같은 함정에 다시 빠진다.

1. **보존:** 상충하는 진술 전부를 출처와 함께 병기하라.
2. **분류:** 통설(다수 학술 출처 지지)과 이설(소수·단일 출처)을 구분하라. 단, 다수가 같은 뿌리의 복제일 수 있으니 3절의 독립성 검사를 먼저 하라.
3. **채택 권고:** cross-validator는 출처 위계(1차 > 학술 > 기관 > 백과)에 따라 채택 권고안을 제시하되, 권고 사유를 명시하라. 권고는 권고일 뿐 — 상충 자체는 conflicts.md에 영구 보존된다.
4. **해소 불가:** 위계가 같은 출처끼리 상충하면 억지로 결정하지 마라. `disputed`로 표기하고 양론 병기를 권고하라. 사이트에 "기록마다 다르다"고 쓰는 것이 틀린 단정보다 낫다.
5. **회부:** 상충이 사건 날짜에 관한 것이면 timeline-analyst에게, 사실 자체에 관한 것이면 synthesis-editor에게 채택 권고와 함께 전달하라.

### conflicts.md 기록 형식

cross-validator는 상충 1건당 다음 요소를 갖춘 항목으로 conflicts.md에 기록하라. 형식이 통일되어야 timeline-analyst와 synthesis-editor가 기계적으로 소비할 수 있다.

- **상충 ID와 대상:** `cfl-{일련번호}` + 관련 claim_id/event_id
- **진술 A / 진술 B(/C...):** 각 진술과 출처(위계 단계 포함)
- **독립성 검사 결과:** 어느 진술들이 같은 뿌리인지
- **통설/이설 판정과 채택 권고:** 권고안 + 위계 기반 사유. 해소 불가면 "양론 병기 권고"
- **재검토 기록:** 자기 권고를 1회 재검토한 결과 (유지/수정)

## 5. 검증 워크플로우

1. **수집:** Phase 1 events.json 전체와 report.md를 읽고, 사건 레코드의 summary·detail에서 주장을 추출하라.
2. **분해:** 1절 기준으로 주장 단위 분해. claim마다 `event_ids`로 출발점을 보존하라.
3. **독립 출처 확인:** claim별로 독립 출처 2개 확인을 시도하라(3절). 레코드에 이미 달린 출처도 독립성을 재검사하라 — 연구원이 같은 뿌리 출처 2개를 단 경우가 있다.
4. **등급 부여:** 2절 기준표로 판정하고 `grade_reason`을 기록하라.
5. **상충 회부:** 확인 과정에서 상충이 드러나면 4절 절차로 cross-validator에 회부하라. 등급은 상충 해소 권고가 나올 때까지 잠정 D로 둔다.
6. **재검증:** 보완 조사나 새 사료 발굴로 입력이 바뀌면 영향받는 claim만 재판정하되, 등급이 바뀐 claim은 `grade_reason`에 변경 사유와 이전 등급을 남겨라. 변경 이력이 없으면 하류 소비자(synthesis-editor)가 어느 버전 기준으로 작업했는지 알 수 없다.

## 6. claims_register.json 스키마

fact-checker의 산출물 `_workspace/02_verified/claims_register.json`은 다음 스키마의 배열이다.

```json
{
  "claim_id": "clm-{일련번호}",          // 주장 고유 id. 전역 유일
  "text": "검증 가능한 단일 명제 형태의 주장 문장",
  "event_ids": ["evt-..."],             // 이 주장이 추출된 사건 레코드 id (1개 이상)
  "grade": "A|B|C|D",                   // 2절 기준표에 따른 신뢰도 등급
  "grade_reason": "등급 판정 사유 — 독립 출처 확인 결과, 사료 비판 반영 사항, 확인 실패 사실 포함",
  "sources": [                          // 검증에 사용한 출처 (사건 레코드의 sources와 동일 형식)
    {
      "type": "primary|academic|institutional|encyclopedia|web",
      "title": "출처 제목",
      "locator": "권·호·면 또는 문서번호",
      "url": "접근 URL 또는 null"
    }
  ]
}
```

운영 규칙:

- 검증 불가 주장도 D등급으로 등재하라. 레지스터에서 빠진 주장은 "검증을 통과한 것"과 구별되지 않는다.
- 하나의 사건 레코드에서 여러 claim이 나오는 것이 정상이다. 역으로 모든 사건 레코드의 핵심 주장이 레지스터에 있어야 한다 — 누락된 사건은 검증되지 않은 채 하류로 흘러간다.
- synthesis-editor는 이 레지스터를 소비하는 입장에서 사용한다: 본문은 A·B등급만, C등급은 "~로 전해진다" 명시, D등급은 제외 목록으로.

## 산출물 검증

제출 전 다음을 확인하라.

1. claims_register.json이 유효한 JSON이고 `claim_id`가 전역 유일한가.
2. 모든 claim에 `grade`와 비어 있지 않은 `grade_reason`이 있는가.
3. 모든 `event_ids`가 Phase 1 events.json들에 실재하는가 (끊어진 참조 0건) — 스크립트로 대조하라.
4. A등급 claim의 sources에 `type: "primary"` 출처가 실제로 포함되어 있는가 — A인데 1차 사료가 없으면 등급 오류다.
5. 독립 출처 2개 확인에 실패한 claim의 `grade_reason`에 실패 사실이 기록되어 있는가.
6. 상충이 발견된 claim이 conflicts.md에도 등재되어 있는가 — 레지스터와 conflicts.md의 양방향 일치를 확인하라.
